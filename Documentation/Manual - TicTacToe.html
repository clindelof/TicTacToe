Tic-Tac-Toe - Lab

   

Objective


    You are given an incomplete codebase that will, when completed, play multiple games of Tic-Tac-Toe using multiple threads.  Your objective is to control the flow of each game using condition variables. Don't worry about implementing the Log and LogSync functionality until you've already completed everything else.

Instructions

    The program that is delivered to you takes two arguments in the following form:

Usage: TicTacToe gameCount playerCount

Arguments:
    gameCount                    Number of games.
    playerCount                  Number of players.

    A typical game of tic-tac-toe consists of 9 tiles in a 3x3 grid and has two players. One player is 'X' and the other player is 'O'. The 'X' player picks a spot on the board and puts an 'X' on it. Next, the 'O' player picks an unoccupied spot on the board and places an 'O'. This repeats until a player has three of their tokens in a row either horizontally, diagonally or vertically. If all tiles on the 3x3 grid are taken and there are no winners then the game results in a tie.

   In our game of tic-tac-toe each player will be running on a separate thread. Each player thread will have to wait for all of the other player threads to be created. Once all players have been created the main thread will signal all waiting players to wake up and begin executing their game logic. After all player threads have completed executing their game logic they will need to let main know that they're done. At this point all games have been completed.

  Once the 'starting gun' has been fired each player will begin to play games. The player will go through the entire pool of games and try to play each game. The game will be skipped if it already has 2 players, otherwise the player will join that game. The first player that joins a game will be assigned 'O' and it will have to wait for the second player to join. The second player that joins the game will be assigned 'X' and will begin playing the game. At this point in time player 'O' is waiting and player 'X' has just started to play the game.

  During the main game loop one player will always be waiting and the other will be playing. The first player, player 'X', will pick a random spot on the board and place their token on it. 

  • If the last move didn't win the game and there are still valid moves then the current player will wake up the other player and then go into a waiting state. The other player will pick a random spot on a board and so on.
  • If the move was a winning move or a draw then the current player will wake up the other player and leave the game. The second player will wake up, notice the game is over and leave the game.
